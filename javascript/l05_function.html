자바스크립트의 함수

실행의 집합의 가장 작은 단위로 설계도(prototype)에 해당한다.

  • 함수는 선언할 때 function을 꼭 작성한다.
  • 함수의 이름 뒤에는 (매개변수)를 작성해야한다.
  • {} 안에는 실행을 작성한다.
  • 실행의 결과나 상태는 return으로 반환해 줄 수 있다.
  • return이 실행되면 함수는 종료된다.
  • function name(param,param2..){실행; 실행2;... return 상태}
  • 함수는 자동으로 실행되지 않는다.(변수처럼 선언된 것이다.)
  • 함수를 실행하기 위해서는 "함수이름();" 방식으로 호출 해야한다.
  • 함수를 실행할 때는 매개변수의 규칙을 따라한다. 예) (sum();=>NaN) == sum(undefined,undefined), sum(20,30);
  • 매개변수는 전역변수와 같은 이름을 사용할 수 있다.
  • 함수 안에 변수는 매개변수와 이름이 달라야 한다.
  • 함수의 이름은 전역변수와 예약어와 같아서는 안되다.
  • 자바스크립트는 함수의 오버로딩을 지원하지 않는다.

자바스크립트의 함수가 가진 특별한 점

  • 함수를 변수(const)가 참조할 수 있다.
  • 함수를 선언하면 전역으로 hoisting이 일어나서 어디서든 참조가능 하다.
  • 변수가 참조하면서 선언된 함수는 hoisting이 일어나지 않는다.
  • 매개변수의 규칙을 따르지 않아도 함수는 실행되고 매개변수는 자동으로 undefined가 된다.
  • 매개변수에 초기값을 지정할 수 있다.(오버로딩을 대신한다)
  • new 연산자로 함수를 호출하면 객체가된다.(prototype을 생성한다)
  • 매개변수를 ...args 선언하면 매개변수의 개수를 제한하지 않는다. sum(1,2,3,4,5,6,7,.....)

const로 참조하는 함수를 사용하는 이유

  • const로 참조하면서 선언된 함수는 참조할 때 메모리에 저장된다.
  • 정상적인 방법으로 선언된 함수는 자바스크립트를 분석할때(문서가 화면에 로딩될때) 메모리에 저장된다.
  • 만약 호이스팅 되어야할 함수가 수천개가 된다면 화면에 로딩이 느려질 수 있다.
  • 브라우저가 화면(document)을 로딩할 때 할 일이 많기 때문에 참조되는 함수로 일을 줄일 수 있다.
