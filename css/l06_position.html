css postion으로 layout을 구성하자

  • static(default) : 모든 요소(노드)의 위치를 정의(document를 기준으로)
  • relative : 자식 노드의 기준점 변경
  • absolute : relative인 부모를 기준으로 절대적 위치
  • fixed : viewport를 기준으로 절대적 위치
  • sticky :

position relative

  • static 위치를 기준으로 top,bottom,left right 이동 가능
  • 다른 노드와 간섭한다.(위치가 이동되면 복사되어서 출력된 거 처럼 기존에 노드 위치에 노드는 존재하지면 출력된 노드는 간섭하지 않는다)
  • z-index를 지정할 수 있다.

position absolute

  • 조상(부모)의 relative를 기준점으로 노드를 top,bottom,left,right로 이동할 수 있다.
  • 만약 조상 중에 relative가 없으면 body를 기준으로 노드를 이동 시킬 수 있다.
  • width:100% default인 block이 absolute가 되면 기준점(top,left)이 되는 조상이 존재하지만 포함 관계가 아니기 때문에 %를 인지할 부모 or 조상이 없다. (width:fit-contnet)
  • absolute 각 노드는 독자적인 존재다. 때문에 각 노드는 서로 간섭하지 않고 간섭하지 않기 위해 z-index가 존재한다.

position fixed

  • 노드 개체가 absolute 처럼 독자적으로 존재한다.(다른 노드에 간섭하지 않는다)
  • 뷰포트(viewport)를 기준으로 노드를 배치한다.(브라우저의 scroll을 따라다닌다.)
  • 독자적인 노드기 때문에 넓이가 fit-content고 z-index가 존재한다.

position sticky

  • sticky 지정된 노드는 평소에 static 위치에 존재하다가 스크롤의 의해 view가 바뀌면(플로우 루트에 의해 지정된 임계값[top:0]을 넘으면 마치 fixed 된것 처럼 화면 또는 보모 노드에 고정된다.)
  • sticky 는 최신 등록된 프로퍼티기 때문에 -webkit-sticky 써야한다.
